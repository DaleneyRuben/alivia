# Initial credentials via emailed setup link

When the founder provisions a Doctor or Assistant account (see [0005](0005-concierge-doctor-onboarding.md), [0013](0013-assistant-accounts-founder-provisioned.md)), the account is created without a password. The system emails the account holder a one-time setup link where they choose their own password; admin-driven password resets (see the Admin practice detail flow) reuse the same mechanism. This means v1 does include transactional email infrastructure — a deliberate addition to the stack in [0012](0012-mvp-tech-stack.md), scoped to credential delivery only (patient-facing messaging stays on manual WhatsApp per [0003](0003-whatsapp-via-manual-links.md)). Alternatives rejected: a founder-typed temporary password (weak/reused passwords, and the founder learns the credential) and a generated password shown once to the founder (same secret-sharing problem, plus a manual WhatsApp relay step). Schema-wise, `User.passwordHash` is nullable until the link is used, with a single active reset token per account.
