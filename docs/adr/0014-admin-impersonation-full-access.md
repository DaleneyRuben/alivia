# Admin impersonation has full access, including Medical History

The founder's Admin view can impersonate any Doctor or Assistant account (log in as them) for support purposes. This is a deliberate, founder-level exception to the Medical History siloing guarantee in [0011](0011-medical-history-siloed-per-doctor.md), which promises no _other Doctor_ ever sees a given Doctor's Medical History for a shared Patient — that guarantee still holds between Doctors. Impersonation was scoped to full access rather than carving out Medical History because a support request that touches a Patient's Appointment often can't be cleanly separated from their Medical History (e.g. a Doctor asking for help after marking the wrong visit Attended), and a partial-access impersonation mode would be more complex to build for a case that only the founder, not another Doctor, can trigger.
